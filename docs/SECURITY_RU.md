# Безопасность

## Никогда не хранить в git
- реальные пароли
- session cookies
- access tokens
- recovery codes
- OTP secrets

## Хранить локально
Используй:
- `.env`
- локальный password manager
- secrets manager, если позже понадобится

## Что делать с логином для платформ
В knowledge можно хранить:
- какой email использовать,
- для каких платформ он используется,
- нужно ли отдельное имя профиля,
- заметки по onboarding.

Но сам пароль должен жить только в `.env`.

## CAPTCHA / OTP / verification
Агент не должен:
- обходить CAPTCHA,
- ломать rate limits,
- подделывать верификацию,
- придумывать подтверждения.

Правильное поведение:
1. сохранить checkpoint,
2. записать blocker,
3. попросить ручное действие,
4. потом продолжить.

## Рекомендация
Если пароль был отправлен в чат и он реально активен, лучше сменить его и сохранить новый только локально.
